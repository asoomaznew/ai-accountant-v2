# Deployment & Security

Operational notes for the AI-Accountant repository. Last updated: 2026-07-12.

## 1. Git remotes

Two remotes are configured for the local clone:

| Remote | URL | Purpose |
|---|---|---|
| `origin` | `git@github.com:asoomaznew/ai-accountant-app.git` | Primary repo |
| `neworigin` | `git@github.com:asoomaznew/ai-accountant-v2.git` | v2 mirror / staging |

Both use **SSH** (no password prompts). Public key registered with title
`hermes-agent-YYYY-MM-DD` in https://github.com/settings/keys.

To rotate: `ssh-keygen -t ed25519 -C "new-label" -f ~/.ssh/id_ed25519_new`,
then add the new public key to GitHub and remove the old one.

## 2. Authentication

- **HTTPS password auth: disabled at GitHub level (2FA on).**
- **SSH key (ed25519):** preferred. Already configured.
- **Personal Access Token (PAT):** for CI or one-off API calls only.
  Scopes: `repo`, `workflow`. Keep in macOS keychain, not in `.git/config`.

To add a new PAT to keychain:
```bash
security add-internet-password -s github.com -r "htps" -a "asoomaznew" -w "$TOKEN"
```

## 3. Local dev servers

Frontend (Vite, port 3000):
```bash
cd frontend && unset PYTHONPATH && npm run dev
```

Backend (FastAPI, port 8000):
```bash
cd backend && unset PYTHONPATH && ./venv/bin/python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

The `unset PYTHONPATH` is required because Python 3.14 in the venv must
not see the parent process's `PYTHONPATH` (which points to a Python 3.11
hermes-agent venv — would cause `pydantic_core` ABI mismatch).

## 4. Vercel deployment

Vercel URL pattern: `https://ai-accountant-<hash>-haithams-projects-<id>.vercel.app/`
is reserved for future deploys. **Not yet provisioned** — see
`# Step 7` below.

To deploy:
1. Enable 2FA on GitHub (must be done in browser).
2. Generate a fresh PAT with `repo` + `write:packages` scopes.
3. `npm i -g vercel && vercel login && vercel deploy --prod`
4. For backend, use Railway / Render / Fly.io (Vercel does not host
   Python directly).

## 5. Required env vars

Frontend (`frontend/.env`):
```ini
VITE_GEMINI_API_KEY=...           # optional, dev only — prefer proxy
VITE_BACKEND_URL=http://localhost:8000
VITE_BACKEND_TOKEN=               # dev: empty (bypass); prod: real token
```

Backend (`backend/.env`):
```ini
ACCOUNTANT_DB_PATH=./accounting.db
GOOGLE_API_KEY=...
ALLOWED_EMAILS=["your@email.com"]
```

`backend/.env` is gitignored — never commit secrets.

## 6. Persistent audit log

Every journal entry generated by the pipeline can be persisted via
`backend/utils/database.py`. The module auto-creates `backend/accounting.db`
on first call and exposes an async API:

```python
from backend.utils.database import save_journal_entry, list_entries
await save_journal_entry({...}, request_id=...)
rows = await list_entries(account_no="KIBAA-2380")
```

To enable in `main.py`: add `await lifespan_setup()` to startup and
`await lifespan_teardown()` to shutdown.
